CR-Fix 1.2.1 - Final Update

This is the last release of CR-Fix. Crunchyroll created a new frontend and moved it out of beta as of 2022-10-19. The plugin isn't compatibel with the new frontend and I don't intended to rewrite everything. Once the old design is fully gone with the planned shutdown of some features in November 2022 there's probably no reason to keep the extension installed.
The good news is some of the fixes this extension still offered in 1.2 are no longer necessary by now. (Unfortunately, some new bugs popped up. E.g. even before the new design - wrt subtitle scaling making them blurred on most HiDPI displays, but fixing them from an extension would be too much effort. A simple workaround is to zoom-out of the page to increase the sub resolution. This effect persists on fullscreening the player.)

Improvements:

  • Fix Update notification showing up on every browser update instead of only updates of the extension itself.

CR-Fix 1.2 - Update

New Feaures:

Customisable navbar
Re-Add the 'Forum' tab to the navbar and/or remove tabs you don't need!
Startpage: Top Simulcast
Move the simulcast section on the startpage to the top, directly below the news carousel. (Moves it above the news section)
Allows you to quickly get an overview of the current releases.
Experimental: Custom SegmentId-Event
Useful in some cases where the original event does not get triggered properly. (Prevents the player from showing up)
If you don't have problems with the player showing up, than you don't need this.

Go to settings and give it a try!

Other Improvements:

  • Improved sidebar positioning for preview episodes and when logged in.
  • Fixed bug which reversed logging priority
  • Minor clarifications on setting page

Disabled Features:

Unfortunately I had to disable the quality and language selection as an update of the Crunchyroll player broke them and they became unusable. I couldn't find a reasonable way to restore the functionality. If you feel like you can do better or just want to know a bit more about the restrictions, check here for more details.

Another setting that had to go is the experimental "Request DRM-free streams". Crunchyroll does no longer acknowledge this cookie, but it now falls back to a sane DRM-free HLS stream by default, when WIdevine is not supported on the target device. So there's no need for this setting anymore anyway.

  • Temporarily(?) disabled quality and language selection
  • Removed "Ex: Request DRM-free streams" setting
